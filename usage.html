Usage

Once the backend and frontend services are running, you can interact with the application through the web interface or directly via the API. The API endpoint for scoring is /score.

Submit a Credit Application

To obtain a credit score via the API, send a POST request to the /score endpoint with a JSON body containing the applicantâ€™s information. Missing values can be represented by empty strings. Here is an example payload: 

{
  "Name": "Jane Smith",
  "ssn": "123-45-6789",
  "Age": "34",
  "Occupation": "Engineer",
  "Annual_Income": "120000",
  "Monthly_Inhand_Salary": "9000",
  "Num_Bank_Accounts": "3",
  "Num_Credit_Card": "2",
  "Interest_Rate": "12",
  "Num_of_Loan": "1",
  "Type_of_Loan": "home",
  "Delay_from_due_date": "0",
  "Num_of_Delayed_Payment": "0",
  "Credit_Mix": "good",
  "Outstanding_Debt": "4000",
  "Credit_Utilization_Ratio": "25",
  "Credit_History_Age": "5 Years",
  "Total_EMI_per_month": "200"
}

The API responds with a JSON object containing a status (ok, flagged or rejected), an estimated credit score and component scores (repayment, utilization, outstanding and inquiries), a risk summary generated by the LLM, a list of recommendations and any anomaly score or recommendation returned by the external service.

Example response:

{
  "status": "ok",
  "credit_score_estimate": 765,
  "repayment": 28,
  "utilization": 20,
  "outstanding": 26,
  "inquiries": 28,
  "summary": "... markdown summary ...",
  "recommendations": ["Reduce credit utilization below 30%", "Maintain payment schedule"],
  "anomaly_score": 0.01
}
